a case study by josh kindla

SA Humane Society

My Contributions

I acted as project manager working with two other UX designers and researchers for this project. I was involved in every process from user research, defining personas, flows and wireframing, to prototyping and UI design.

The Problem

The San Antonio Humane Society website lacks current pet information, and users have found it difficult to accurately search for pet preferences which has contributed to a decrease in adoption numbers.

The Solution

We put interactive adoption information tabs directly on the hero page, provided search filters, and updated pictures and pet information which will help users learn more about their prospective pet.

Research Process

We conducted research by conducting thought out user interviews, user surveys through Google Forums, and conducting a competitor analysis (below) on other websites with similar goals. We learned that users would greatly value pet search and filters, more updated info on pets, and less paperwork. We observed some websites such as “Paws Ranch Rescue”:, and “Animal Cares Service” do well at providing Pet Finder Tools, and other tools that help the user through the adoption process.

Desining the Flow

Our user flow depicts the path a user will take in order to reserve a visit and ultimately adopt a pet. We made sure to keep this flow as simple as possible. Our Storyboard depicts our User’s adoption journey using our improved redesign.
